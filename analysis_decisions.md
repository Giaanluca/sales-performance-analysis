# Analysis Decisions and Reasoning

## Purpose
This document explains the reasoning behind the analytical decisions in this project.  
The goal is to justify why certain variables and visualizations were chosen.

# 1. Variable Selection
The analysis focused on variables directly related to the business question:

- Product, Category, Revenue/Sales Value, Customer, Transaction details

These were prioritized to identify top performers, revenue patterns, and lower-performing segments.  
Variables less relevant to this question were intentionally not analyzed to keep the analysis focused.

# 2. Visualization Choices

### Bar Charts
- Used for comparing categories and products side by side.  
- Easier to rank items and identify top/low performers than pie charts.  
- More suitable than trend charts because the analysis was about performance comparison, not temporal evolution.

### Scatter Plots
- Used to show the distribution of variables and identify patterns, dispersion, and outliers.  
- Provides a more complete picture than summary metrics alone.

# 3. Analytical Approach
Decisions were guided by the principle: *nalyze what matters for the business question.  
The value of the project lies not only in running code or creating charts, but in choosing variables and visualizations intentionally to produce meaningful insights.

# 4. Reflection
Each choice — variable selection, chart type, and analysis approach — aimed to keep the analysis:

- Relevant
- Clear
- Aligned with the business question

The goal was to make the results understandable and actionable, not just to show technical skills.
